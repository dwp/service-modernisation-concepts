Did we test it?

These pages were tested with 10 users in round 3 of research.

Detailed research summary

  • Decided against reanalysing the data as it is very likely that this journey already exists elsewhere and are not specific to multiple benefits. If this has not happened yet, it should be worked with wireframes that were already tested and used – for instance in Universal Credit or other benefits where people use two-factor-authentication.

Considerations for further research and design work

  • Base designs on standard government or DWP patterns for multi-factor authentication.
  • Explore options for displaying the full phone number or email address to the user to make sure we are sending the code to the correct place, especially because this information appears in the profile page a few screens before.
  • Reconsider whether it is necessary to partially hide the phone number if it appears a few screens earlier.
  • At the bottom of the confirmation page, review the use of the primary button component. This is the end of the journey so it is not standard to present the user with a Continue button at this point. Base the design on standard government for confirmation pages.
