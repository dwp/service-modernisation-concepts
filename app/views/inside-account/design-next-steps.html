Design next steps

Possible areas for design focus:

General

  • Understand how the header aligns with the OneLogin header.
  • Understand how the OneLogin account aligns to this account.
  • Explore options for how we reassure the user about where they are.
  • Carry out some desk research into other 'accounts' that exist in DWP and across government that these users may be familiar with.
  • Ensure the top priority information is on each screen.
  • Ensure the information hierarchy is clear on each screen.
  • Ensure the information is laid out in the most intuitive way.
  • Explore how we can make numeric content work for users who struggle to understand numbers (e.g. users with dyscalulia), as well as work for users who may have dyslexia.

Home page

  • Explore how we can allow a user to track applications and can create a consistent experience across pages and benefits.
  • Ensure any design for progress tracking meets design guidelines.
  • Understand what users currently receive in relation to their benefit payments (letters, text messages, emails ando so on).
  • Explore how we can best show payment information to users (and align the information to what arrives in their bank account).

Apply and track page

  • Explore how we can allow a user to track applications and can create a consistent experience across pages and benefits.
  • Ensure any design for progress tracking meets design guidelines.
  • Understand when users expect to be updated on applications they have submitted.
  • Explore how we can best show payment information to users (and align the information to what arrives in their bank account).

Payments page

  • Explore how we can allow a user to track applications and can create a consistent experience across pages and benefits.
  • Understand when users expect to be updated on applications they have submitted.
  • Explore how we can best show payment information to users (and align the information to what arrives in their bank account).
  • In 'Your benefits', explore how we can make it easy for users to access and understand information about which benefits they are receiving.
  • In 'Other services', make it clear to the user which pages/support belong to DWP and which are external and approved for us to link to.
  • In 'Other services', explore options for personalising links.

Your application page

  • Explore how we can allow a user to track applications and can create a consistent experience across pages and benefits.
  • Ensure any design for progress tracking meets design guidelines.
  • Understand when users expect to be updated on applications they have submitted.

Support page

  • Understand the expectations users have about accessing support.
  • Test the language users expect in relation to accessing support.
  • Explore where in the account users expect to go to for support.
